# Architecture

Audio Track Editor is organized around a small, testable core and optional heavy
adapters. The MVP is timeline-based speaker muting inside one selected
audio/language track at a time. The app should remain usable for metadata
inspection and project editing even when ML dependencies have not been installed
yet.

## Components

- `audio_track_editor.cli` exposes the `ate` command for doctor, analyze, and
  export workflows.
- `audio_track_editor.gui` contains the PySide6 desktop shell.
- `audio_track_editor.config` reads ignored local settings from `.env`,
  `.ate.local.toml`, and environment variables.
- `audio_track_editor.schemas` defines project, stream, speaker, and segment
  data structures.
- `audio_track_editor.media` owns ffprobe parsing and FFmpeg command building.
- `audio_track_editor.analysis` coordinates media inspection now and will call
  diarization/alignment/separation adapters as they are added.
- `audio_track_editor.timeline` applies confidence fallback rules, segment
  merging, and crossfade planning.
- `audio_track_editor.muting` renders a modified WAV by applying padded
  fade-aware mute envelopes for selected speakers.
- `audio_track_editor.renderer` writes generated subtitle cues and exports either
  muted WAV or remuxed video with the modified audio.

## Project Schema

Project files are JSON so users can inspect and recover them easily. A project
stores:

- source media path
- selected base audio stream
- video/audio/subtitle stream metadata
- detected speaker IDs and optional human labels
- speaker mute toggles
- segment start/end times
- source and target audio streams
- confidence, overlap, and subtitle fallback flags
- render settings for fade, padding, merge gap, and mute gain

The schema is intentionally simple for the first version. Future migrations
should preserve old project loading and add a migration command before breaking
existing `.ateproj.json` files.

## Profile Schema

Reusable show/character profiles are planned as separate JSON files. A profile
will map stable speaker/character labels to preferred audio and subtitle
behavior. The project schema already includes speaker labels and preferences so
the profile layer can be added without changing the renderer contract.

## Analysis Pipeline

The intended full pipeline is:

1. Use ffprobe to inspect container streams.
2. Extract analysis WAVs per relevant audio stream.
3. Run pyannote `speaker-diarization-community-1` locally to answer who spoke
   when.
4. Run ASR/alignment to connect timing to text and subtitle cues.
5. Run vocal/background separation to create a base bed.
6. Attempt experimental speaker-conditioned separation on overlap-heavy regions.
7. Write project segments with confidence and overlap metadata.

The current scaffold implements step 1 and writes a placeholder review segment.
This lets the CLI, GUI, project schema, tests, and export path settle before the
large models are integrated.

## Local And Offline Model Policy

`ATE_OFFLINE_MODE=true` is the default. Model adapters should treat that as a
hard signal to use local paths/caches only during analysis and export. Tokens,
when configured, are for explicit setup/download flows and not for normal media
processing.

`models/` is a cache location, not a source folder developers need to populate
manually. It can be copied to another PC or redirected in `.env`.

## Rendering And Fallback

The MVP renderer extracts a high-quality 48 kHz stereo WAV for the selected
track, collects all segments belonging to muted speakers, applies pre/post
padding, merges nearby regions, and writes an output WAV with short fade down/up
around muted sections.

This is not true voice-only removal. Background music, sound effects, and other
overlapping speakers may also be muted during selected regions. Later versions
can add dialogue stem separation and target-speaker suppression.

## Failure Modes

- Missing FFmpeg: `ate doctor` reports warning and setup docs explain how to set
  explicit paths.
- Missing model dependencies: metadata workflows still run; full analysis should
  show actionable setup guidance.
- Low confidence or overlaps: project segments are flagged and subtitles are
  generated by default.
- Copyrighted media: ignored local folders are documented and excluded from git.
